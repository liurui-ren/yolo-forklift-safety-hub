# 叉车作业人车互斥报警系统 — 云端监控平台项目报告

## 一、项目概述

叉车作业人车互斥报警系统是一套面向工业现场的实时安全监控系统，旨在解决厂内叉车与行人混行区域的安全隐患问题。系统采用 **"端-云"协同架构**，本报告聚焦于**云端监控平台**的实现情况。

云端平台基于 **Flask + Socket.IO + MQTT + SQLite** 技术栈构建，采用 **Vue 3 单页应用（SPA）** 作为前端界面，实现设备状态实时监控、报警数据管理、历史追溯与可视化分析等核心功能。

## 二、系统架构

### 2.1 技术架构

系统采用分层架构设计，数据流如下：

```
MQTT Broker → mqtt_client.py → db.py (SQLite) → Socket.IO → Vue 3 前端
                    ↓
              logger.py (文件 + 数据库双写)
```

### 2.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | Flask + Flask-SocketIO | REST API 与实时通信 |
| 消息中间件 | paho-mqtt (MQTT 3.1.1) | 设备数据接入 |
| 前端框架 | Vue 3 (Composition API) + Vite | 单页应用，组件化开发 |
| 数据可视化 | ECharts | 折线图、散点图、地图 |
| 数据库 | SQLite3 (WAL 模式) | 轻量级并发读写 |
| 通信协议 | MQTT + WebSocket + REST API | 多协议协同 |

### 2.3 前端架构

- **框架**：Vue 3 (Composition API) + Vue Router + Vite
- **UI 风格**：Glassmorphism（毛玻璃）+ Static Aurora Gradient（静态极光渐变背景）
- **字体**：Outfit（展示）+ DM Sans（正文）+ JetBrains Mono（等宽）
- **页面组件**：
  - **Dashboard（仪表盘）**：数字孪生地图 + 报警趋势图 + KPI 数据卡片
  - **Devices（设备列表）**：状态筛选 + 设备详情 + 报警图片查看
  - **Logs（业务日志）**：分页查询 + 多条件筛选（级别/分类/设备）
- **可复用组件**：
  - LineChart.vue（ECharts 折线图，支持自定义数据、颜色、线宽）
  - AuroraBackground.vue（静态极光渐变背景）
  - SidebarNav.vue（左侧导航栏，含 logo）

## 三、已实现功能

### 3.1 设备管理

- **设备注册与状态监控**：自动识别设备在线/离线状态，超时自动判定离线
- **实时状态推送**：通过 Socket.IO 推送 `device_update` 事件，前端无刷新更新
- **设备位置追踪**：支持设备坐标存储，数字孪生地图实时展示设备位置与状态
- **设备详情查询**：查看设备报警历史、报警时长统计（平均/最长/当前持续时长）

### 3.2 报警系统

- **MQTT 报警接入**：订阅 `factory/forklift/+/alarm` 主题，实时消费设备报警消息
- **报警计数防重复**：仅在"正常→报警"状态跃迁时计数+1，避免重复统计
- **报警图片上传**：支持 MQTT+HTTP 混合方案批量上传报警图片，设备端先通过 HTTP 上传获取 URL，再通过 MQTT 携带 URL 元数据
- **报警时长统计**：记录每次报警会话的起止时间，支持平均时长、最长时长、当前持续时长统计

### 3.3 数据可视化

- **数字孪生地图**：基于厂区平面图，实时展示叉车位置（散点图），按状态着色（正常/报警/离线）
- **报警趋势图**：24 小时报警次数折线图，支持今日/昨日对比
- **KPI 数据看板**：设备总数、在线设备数、当前报警数实时展示
- **设备状态列表**：支持按在线状态、区域、设备 ID 多维度筛选

### 3.4 日志系统

- **统一日志表**：支持 ops（运维）、biz（业务）、sec（安全）三分类
- **日志查询**：分页查询，支持按日志级别、分类、设备 ID 筛选
- **双写机制**：日志同时写入文件与 SQLite 数据库

### 3.5 安全认证

- **REST API 鉴权**：支持 `Authorization: Bearer <token>` 或 `X-Auth-Token` 请求头
- **Socket.IO 鉴权**：支持 auth payload 或 query 参数传递 token
- **开发模式**：`AUTH_ENABLED` 默认关闭，便于调试

## 四、核心特性

### 4.1 实时性

- MQTT 消息实时消费，延迟 < 100ms
- Socket.IO WebSocket 推送，前端状态更新 < 200ms
- 后台离线检测线程每 5 秒巡检一次，10 秒超时判定

### 4.2 并发可靠性

- SQLite WAL 模式支持并发读写
- 数据库 busy_timeout 超时机制（默认 5000ms）
- 后台任务使用 `socketio.start_background_task()` 安全调度

### 4.3 用户体验

- Vue 3 SPA 无刷新页面切换
- 毛玻璃 + 静态极光渐变背景现代 UI 设计
- 响应式布局，适配桌面与移动端
- ECharts 图表支持交互（悬停提示、悬停加粗）
- 统一的左侧导航栏 + logo 品牌标识

## 五、应用场景

- 厂内叉车与行人混行区域的实时安全监控
- 设备在线状态与报警状态统一看板
- 报警记录留痕、趋势分析与运维排障
- 多设备位置追踪与数字孪生地图可视化
